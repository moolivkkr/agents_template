# Forcepoint DLP Policy Authoring -- Quickstart

> Fastest path from zero to a working DLP policy in Forcepoint DLP.

---

## Prerequisites

Before you start, confirm:
- [ ] Forcepoint Security Manager is installed and accessible
- [ ] You have Administrator credentials for the Data Security module
- [ ] At least one Forcepoint DLP component is deployed (endpoint agent, network protector, or email gateway)
- [ ] DNS and network connectivity between the management server and DLP components is verified
- [ ] A successful initial deployment (Deploy button, green status) has been completed at least once

See [prerequisites.md](prerequisites.md) for the full dependency chain.

---

## Path A: Predefined Policy (5 Minutes)

This is the recommended starting point. Forcepoint ships 1,800+ predefined classifiers and policy templates.

### Step 1: Open Policy Management

1. Log in to Forcepoint Security Manager
2. Navigate to **Main > Policy Management > DLP Policies**

### Step 2: Enable a Predefined Policy

1. In the policy list, locate the predefined policy you want (e.g., "HIPAA", "PCI-DSS", "GDPR")
2. Click the checkbox next to the policy name to select it
3. Click **Enable** in the toolbar
4. The policy is now active in the configuration (but not yet deployed)

### Step 3: Review the Default Action Plan

1. Click the policy name to open it
2. Navigate to the **Severity & Action** tab
3. Review the default action plan assignments:
   - Low severity: Audit (log only)
   - Medium severity: Audit + Notify
   - High severity: Block
4. For initial deployment, change ALL severity levels to **Audit only** (recommended)
5. Click **OK** to save

### Step 4: Deploy

1. Click the **Deploy** button in the Security Manager toolbar
2. Click **OK** to confirm
3. Wait for all components to show **Success** status
4. Policy is now live and monitoring traffic

### Step 5: Monitor Incidents

1. Navigate to **Main > Reporting > Incident Manager**
2. Review incidents generated by the new policy over the next 3-7 days
3. Evaluate false positive rates
4. When satisfied, return to Step 3 and enable enforcement actions (Block, Quarantine, etc.)

**Total time: ~5 minutes for steps 1-4, then 3-7 days of monitoring before enforcement.**

## Path B: Custom Policy (15 Minutes)

When predefined policies do not cover your specific need.

### Step 1: Create a Content Classifier

1. Navigate to **Main > Policy Management > Content Classifiers**
2. Click **New** to create a classifier
3. Choose classifier type:
   - **Patterns & Phrases** for regex, key phrases, or dictionaries
   - **File Properties** for file type/size rules
   - **Fingerprinting** for document or database matching
4. Configure the classifier (e.g., regex pattern for a custom ID format)
5. Click **OK** to save

### Step 2: Create the Custom Policy

1. Navigate to **Main > Policy Management > DLP Policies**
2. Click **Create custom policy**
3. The 5-tab wizard opens:

**Tab 1 -- General:**
- Enter a policy name (e.g., "Protect Internal Project Data")
- Add a description
- Click **Next**

**Tab 2 -- Condition:**
- Click **Add Condition**
- Select your classifier from Step 1
- If using multiple classifiers, set logic (AND / OR / NOT)
- Click **Next**

**Tab 3 -- Severity & Action:**
- Set threshold levels:
  - Row 1: "Where there are at least 1 match" -> Severity: Low -> Action Plan: Audit
  - Row 2: "Where there are at least 10 matches" -> Severity: Medium -> Action Plan: Audit & Notify
  - Row 3: "Where there are at least 50 matches" -> Severity: High -> Action Plan: Block
- Click **Next**

**Tab 4 -- Source:**
- Leave as "All sources" for now (or select specific user groups/departments)
- Click **Next**

**Tab 5 -- Destination:**
- Leave as "All destinations" for now (or select specific channels)
- Click **Finish**

### Step 3: Deploy

1. Click **Deploy** in the toolbar
2. Confirm and wait for Success status

### Step 4: Monitor and Tune

1. Check incidents after 24-48 hours
2. Adjust thresholds based on false positive rates
3. Narrow source/destination scope as needed
4. Redeploy after changes

## Path C: Clone and Customize (10 Minutes)

Best when a predefined policy is close to what you need but requires modifications.

### Step 1: Clone

1. Navigate to **Main > Policy Management > DLP Policies**
2. Right-click the predefined policy
3. Select **Clone**
4. A copy is created with "(Copy)" appended to the name

### Step 2: Customize

1. Open the cloned policy
2. Rename it (remove "(Copy)", add your organization's naming convention)
3. Modify rules, thresholds, sources, destinations, or action plans as needed
4. Add or remove classifiers from rule conditions
5. Save changes

### Step 3: Disable Original (Optional)

If you want only the customized version active:
1. Select the original predefined policy
2. Click **Disable**

### Step 4: Deploy

1. Click **Deploy** in the toolbar
2. Confirm and monitor

|----------|-----------------|--------|------|

| GDPR compliance | Path A (Predefined) | Enable "GDPR" predefined policy | 5 min |
| Block USB exfiltration | Path B (Custom) | File property classifier + USB destination | 15 min |
| Protect a specific database | Path B (Custom) | Database fingerprint classifier + custom policy | 30 min |
